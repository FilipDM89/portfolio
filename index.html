Over Mij

De eerste keer dat ik in aanraking kwam met web development, was toen ik nog maar 15 jaar oud was. Ik was lid van een bandje en we vonden dat we een website nodig hadden. Niemand van ons wist hoe je een website maakte, dus heb ik die last maar op mijn schouders genomen. Mijn allereerste pagina werd gemaakt met enkel en alleen HTML. Uiteindelijk heb ik verder nooit wat gedaan met die kennis.

Na het behalen van mijn schooldiploma heb ik taalkunde gestudeerd, omdat al m'n leraren me die kant opstuurden, en het mij het meest interesseerde. Heel lang heb ik programmeren intimiderend gevonden. Wanneer mijn studie afgewerkt was, heb ik iets meer dan een half jaar gewerkt in de klantenservice. Dat was niet wat ik wou doen, en even wist ik niet meer wat aan te vangen. Toen herinnerde ik mij die tijd dat ik een website gemaakt had. Een vriend van mij had een carrièreswitch gemaakt naar de IT. Dat was het, dat wou ik ook doen! Het heeft lang geduurd, maar bij Codegorilla vond ik een uitstekende aangelegenheid om een goeie basiskennis aan te leggen. Sindsdien heb ik het gevoel het beter te snappen en haal ik er enorm veel voldoening uit om een mooi werkend product te creëren.

HTML & CSS

HTML en CSS heb ik allebei goed onder de knie. Conceptueel zijn ze niet erg moeilijk om te vatten. Met CSS kan ik dingen vrij snel en makkelijk stylen. Mocht er zich toch een probleem voordoen, kan ik het meestal snel oplossen. Verder ben ik me bekend met het grid-systeem. Dat doe ik meestal met bootstrap, maar in principe ben ik me bewust van andere zoals Semantic UI en Materialize. Tevens ken ik het concept van SaaS.

Javascript

Deze programmeertaal was zwaar om te leren begrijpen, maar ik heb het toch gehaald. Tijdens mijn opleiding bij Codegorilla heb ik daar een goeie basis gelegd. Het was mijn prioriteit om droge Javascript zo uitvoerig mogelijk te snappen. Daarom heb ik tijdens het maken van mijn allereerste eigen app, zo weinig mogelijk opzoekwerk te verrichten en hulp te vragen. Intussen snap ik de grond wel. Eens die basiskennis er was, ben ik me gaan verdiepen in jQuery. Op zich vind ik het handiger en sneller om mee te werken en daarom ook aangenamer. Wat ik graag nog in de toekomst zou willen leren zijn frameworks zoals React, Vue of Angular.

PHP - Laravel & Node.js

Tijdens de bootcamp bij Codegorilla, hebben we Node.js als backendtaal geleerd. De leercurve hiervoor vond ik groot. Als framework daarvan gebruikten we express, dat op zich nogal leeg is. Het is aan de developer om daar het geraamte als het waren aan te leggen. Voor een beginneling vond ik dit zwaar. Bovendien werd Node.js als backendtaal te weinig gebruikt om erop voort te bouwen. Vervolgens ben ik PHP en dan uiteindelijk ook Laravel. De instap hiervoor is kleiner en eenvoudiger. Nu begrijp ik de concepten zoals migrations, models, requests, controllers, etc. Ik weet hoe je o.a. een simpele RESTful API kunt maken volgens het MVC-model. Verder heb ik kennis van relations tussen models en hoe je ze moet aanleggen. Persoonlijk vind ik Laravel veel aangenamer om mee te werken. Leren werken met Laravel heeft me geholpen om belangrijke concepten voor de backend te snappen, en hoe ze in interactie staan met elkaar. Voorts heb ik een basiskennis van MySQL en de phpmyadmin-pagina.
